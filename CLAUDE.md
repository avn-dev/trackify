# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

> **This is the operational guide.** The original product/design brief is preserved at
> [`docs/BUILD_SPEC.md`](docs/BUILD_SPEC.md) — read it for *design intent* (visual language, screen
> inventory, copy), but treat it as **partially outdated**: it specifies Supabase, a single lime
> accent, a custom TabBar, and `Localizable.strings`. The shipped app diverged from all four (see
> [Reality vs. the spec](#reality-vs-the-spec)). When code and `BUILD_SPEC.md` disagree, the code wins.

## What this repo is

Two independent apps plus deployment glue in one repo:

- **`TrackifyApp/`** — native **iOS 17+ SwiftUI** fitness tracker (German UI, du-form). 87 Swift files. Offline-first.
- **`backend/`** — **Laravel 13 + Sanctum** REST API on **SQLite**, shipped as a Docker container to `trackify-api.vision2co.de`. Currently used by the app **for auth only**.
- **`deploy/`** — VPS provisioning + redeploy scripts (host nginx reverse proxy + Cloudflare origin TLS).
- Design source-of-truth assets at the root (`*.jsx`, `Trackify*.html`, `tokens.jsx`, `design_handoff_swift_ios/`) — the React/HTML prototypes the SwiftUI app was ported from. Not built; reference only.

The iOS app and the backend are only loosely coupled: **the app talks to the backend solely for login/register/Apple sign-in.** No workouts, runs, body metrics, lab values, supplements, or peptides are ever synced to the server — they live only in on-device SwiftData.

## iOS app (`TrackifyApp/`)

### Build / test commands

The Xcode project is **generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml`**. `TrackifyApp.xcodeproj` is committed but is a build artifact.

```bash
# Regenerate the project — REQUIRED after adding/removing/renaming any source file.
xcodegen generate            # never hand-edit project.pbxproj; edit project.yml instead

# Build for the simulator (no signing team needed; DEVELOPMENT_TEAM is empty)
xcodebuild build -project TrackifyApp.xcodeproj -scheme TrackifyApp \
  -destination 'generic/platform=iOS Simulator'

# Run the full unit-test bundle (needs a CONCRETE simulator, not 'generic')
xcodebuild test -project TrackifyApp.xcodeproj -scheme TrackifyApp \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro'

# Run a single test:  -only-testing:<bundle>/<class>/<method>
xcodebuild test -project TrackifyApp.xcodeproj -scheme TrackifyApp \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  -only-testing:TrackifyAppTests/TrackifyAppTests/testDuration
```

- Only shared scheme is `TrackifyApp`. Run `xcrun simctl list devices available` to pick a destination (iPhone 17 Pro on iOS 26.x exists here).
- Tests are plain **XCTest** (not Swift Testing); `TrackifyAppTests/TrackifyAppTests.swift` only covers `Formatters` (4 methods: `testFormattersKg/Integer`, `testDuration`, `testPace`). There is no UI/feature test coverage.
- Build settings of note (`project.yml`): bundle id `app.trackify.ios`, iOS 17.0, Swift 5.9, **`SWIFT_STRICT_CONCURRENCY: complete`** (everything must be concurrency-clean), previews enabled.

### Architecture (the parts that span files)

- **DI container — `Data/AppDependencies.swift`.** A `@MainActor @Observable` class holding six repositories (`workouts, runs, body, lab, supplements, peptides`). Built once in `TrackifyApp.swift` from the shared `ModelContainer.mainContext` and injected with `.environment(appDeps)`. Views read it via `@Environment(AppDependencies.self) private var deps` and call e.g. `deps.body.latestMetric(...)`. **`init` wires only the SwiftData impls — there is no runtime flag to swap in Mock/Remote.**
- **Repository pattern.** Protocol in `Data/Repositories/*` (`@MainActor`, methods `async throws`) → SwiftData impl in `Data/Local/SwiftData*Repository.swift` (synchronous `throws` against a `ModelContext`, satisfying the async protocol). **SwiftData is the single source of truth (offline-first).** There is **no** API-backed repository.
- **Previews** use `AppDependencies.mock()` — an *in-memory SwiftData* container, **not** the `Data/Mock/MockRepositories.swift` classes (those are implemented but referenced nowhere — dead code).
- **Models — `Data/Models/`.** `@Model` types: `UserProfile`, `Workout`+`WorkoutSet`+`Exercise`, `Run`, `BodyMetric`, `LabMeasurement`+`LabValue`, `Supplement`+`SupplementIntake`, and the peptide triad `Peptide`+`PeptideVial`+`PeptideInjection`. The production `Schema` is declared in `TrackifyApp.swift`.
- **Navigation.** `RootView` (auth gate; owns `AuthState`, restores session on launch) → `MainTabView` (native SwiftUI `TabView`, 5 tabs: home/train/run/body/me). **Lab, Supplements, and Peptides are pushed *inside* the Körper/Body tab** (`BodyTabView` → `BodyRoute`; Supplements + Peptides share a `PreparateHubView` segmented control). Cross-tab jumps use `NotificationCenter` (`.switchToBodyTab`, `.supplementDeepLink`).
- **Networking/services — `Data/Remote/`.** `APIClient` (URLSession + Bearer token, snake_case JSON, ISO8601), `APIConfig` (base URL), `AuthService` (Sanctum login/register/apple/me/logout), `KeychainService` (single token under `app.trackify.ios.token`). Device services: `HealthKitService` (weight read/write, HR), `Features/Cardio/RunTracker.swift` (CLLocationManager GPS + splits + polyline + live HR), `Core/Util/NotificationScheduler.swift` (per-supplement daily local reminders).
- **Theming — `Core/DesignTokens.swift` + `Core/Theme.swift`.** Raw values in `Palette`/`Typography`/`Spacing`/`Radii`; assembled into a `Theme` struct injected via `@Environment(\.theme)`. `ThemedRoot` is the provider and picks dark/light + overrides the accent from `@AppStorage`. **You must wrap any `.sheet`/`.fullScreenCover` content in `ThemedRoot`** or it loses theming + the selected accent.
- **Training plans are NOT in SwiftData.** `Features/Training/PlanData.swift` persists plans as JSON in `UserDefaults` and bumps a `planStoreVersion` `@AppStorage` int for reactivity — a parallel persistence system.

### iOS conventions

- **All numbers go through `Core/Util/Formatters.swift`** (`de_DE` locale → `72,4`, `18.420`). `Formatters.compact()` chooses integer vs. 1-decimal. Stored values are **SI** (kg / km / sec-per-km); lbs/mi/imperial are display-time conversions gated by `@AppStorage("unitsKg"/"unitsKm")`. Never store non-SI.
- `Typography.display()/number()` auto-apply `.monospacedDigit()`; `title()/body/eyebrow` do not.
- German UI strings are **hardcoded inline** (`Text("Workout starten")`). There is **no** `Localizable.strings`/`.xcstrings` and zero `String(localized:)`. Don't write `String(localized:)` for new strings expecting a catalog.

### Live bugs / traps to know

- **Peptide persistence likely crashes in the real app:** `Peptide`/`PeptideVial`/`PeptideInjection` are registered in `AppDependencies.mock()`'s preview schema but **missing from the production `Schema` in `TrackifyApp.swift`**. Previews work; saving a real peptide may fail. Fix by adding them to the production schema.
- **Geist fonts are not in the repo.** `Info.plist` declares 7 `Geist`/`GeistMono` `.otf` files (`UIAppFonts`) and the code uses `Font.custom("Geist"…)`, but no `.otf` exists — it **silently falls back to the system font**. Download from `vercel.com/font` and add to the target to get the intended look.
- `TrackifyApp.entitlements` is an empty `<dict/>` — no HealthKit/Push/App-Group capabilities are actually provisioned despite the Info.plist usage strings.
- `RunTracker` auto-escalates to `.authorizedAlways` immediately after WhenInUse with **no pre-prompt** (App Review risk).

## Backend (`backend/`)

### Commands (run from `backend/`)

```bash
composer setup            # first-time: install + .env + key:generate + migrate + npm build
composer dev              # full stack: serve + queue:listen + pail + vite (concurrently)
php artisan serve         # API only → http://127.0.0.1:8000
php artisan migrate:fresh --seed   # reset DB; seeds ONE user: test@example.com / "password"
php artisan route:list    # verify the route table + middleware
composer test             # config:clear + artisan test  (if it errors on '@no_additional_args', use the next line)
php artisan test                          # phpunit; DB is forced to sqlite :memory:
php artisan test --filter=ExampleTest     # single test by class/method name
php artisan test --testsuite=Feature      # one suite (Unit | Feature)
```

- **PHP 8.3, Laravel 13, Sanctum 4, SQLite** (`backend/database/database.sqlite`, gitignored data; `.env` gitignored, `.env.example` committed).
- A freshly copied `.env` defaults to `APP_ENV=production`, `APP_DEBUG=false`, and `DB_DATABASE=/var/www/html/database/database.sqlite` (a **container path**). For local dev, point `DB_DATABASE` at the committed `backend/database/database.sqlite` and/or set `APP_ENV=local`. `.env` already exists in this checkout.
- Only example tests exist — `composer test` passing tells you nothing about API correctness.

### Backend architecture

- **No `/api` prefix.** `bootstrap/app.php` sets `apiPrefix: ''`, so despite controllers living in `App\Http\Controllers\Api` and comments saying `/api/...`, the real paths are `/auth/login`, `/workouts`, `/health`, etc. (recent commit: "Remove /api prefix — subdomain is already api."). Health routes: custom `GET /health` and framework `GET /up`. **The `docker-compose.yml` healthcheck hits `/api/health` — likely stale; verify against `/health`.**
- **Auth = Sanctum personal access tokens.** Public `POST /auth/{login,register,apple}` issue `createToken(...)->plainTextToken`; protected routes use `auth:sanctum`. Tokens **never expire** (`config/sanctum.php` expiration `null`).
- **Tenant scoping is the only authz:** every protected action filters through `$request->user()->relation()` (e.g. `->workouts()->findOrFail($id)`), so cross-user access 404s. No policies, no global scopes.
- **API surface** (`routes/api.php`): `workouts` (index/store/show/destroy — no update), `runs` (same), `body-metrics` (index/store/destroy), `lab/measurements` (index/store/show/destroy, nested values), `supplements` (full CRUD + `POST supplements/{id}/intake`).
- **Two tables have no model/controller:** `workout_sets` and `exercises` exist in the migration but are unreachable via the API (set-level data only lives in iOS SwiftData).
- ⚠️ **`POST /auth/apple` is a security skeleton** — it base64-decodes the Apple JWT **without verifying the RS256 signature**. Do not treat as production-secure; add `lcobucci/jwt` (or similar) before relying on it.

## Deployment (`backend/` Docker + `deploy/`)

Single all-in-one container (`php:8.3-fpm` + nginx + supervisor) built from `backend/Dockerfile`; SQLite persisted in a named volume; compose publishes **only** on `127.0.0.1:8002`; a **host nginx** reverse proxy terminates TLS for `trackify-api.vision2co.de` (Cloudflare origin cert) and proxies to `:8002`.

```bash
# Build for the prod arch (zsh here wraps docker build/run with --platform linux/amd64)
cd backend && docker build --platform linux/amd64 -t trackify-api .

# Run locally via compose (APP_KEY in .env.example is EMPTY — must generate or the app boots broken)
cd backend && cp -n .env.example .env \
  && docker compose run --rm app php artisan key:generate --force \
  && docker compose up -d --build      # reach it at http://127.0.0.1:8002

# Production (VPS):
bash /var/www/trackify/deploy/server-setup.sh   # one-time provision (Docker, nginx, clone, key, TLS)
bash /var/www/trackify/deploy/update.sh         # redeploy: git pull + docker compose up -d --build
```

- **`entrypoint.sh` runs `migrate --force` on every container start** and rebuilds config/route/view caches. `update.sh` therefore auto-migrates with no backup — a bad migration ships straight to prod.
- `entrypoint.sh` does **not** run `key:generate`; only `server-setup.sh` does (once). `update.sh` assumes `.env` already exists on the server.
- TLS is inconsistent in the scripts: `deploy/nginx-site.conf` uses a Cloudflare origin cert, but `server-setup.sh` also runs `certbot --nginx` — the Cloudflare path is the current intent; the certbot line is likely stale.

## Reality vs. the spec

`docs/BUILD_SPEC.md` is the design intent; the implementation diverged here:

| BUILD_SPEC says | Reality |
|---|---|
| Supabase (Postgres + Auth + RLS) | Laravel 13 + Sanctum + SQLite, Docker → `trackify-api.vision2co.de` |
| Offline-first **with sync** | Offline-**only**; backend used for auth alone (no domain-data sync) |
| Single lime accent `#c8ff3d`, never a second | User-selectable accent (lime/rose/sky/violet) via `AccentOption` |
| Custom HStack `TabBar` (no native chrome) | Native SwiftUI `TabView` |
| All strings via `Localizable.strings` | German hardcoded inline; no catalog |
| UUID ids (`User`), Supabase repos | Int ids server-side; `UserProfile` model; `SwiftData*Repository` |
| 19 screens / 7 sections | + entire **Peptides** domain, + Profile sub-screens (Goals/Units/Reminders/Privacy/ConnectedDevices/DataExport), + Training PlanEditor/PlanLibrary |
| Hand-maintained Xcode project | **XcodeGen** (`project.yml`); run `xcodegen generate` after file changes |

## Project knowledge base (gbrain)

Durable, non-obvious project knowledge (architecture rationale, decisions/ADRs, gotchas, history) lives in
**`brain-seed/`** (Markdown) and is synced into the local **gbrain** knowledge base. See
[`brain-seed/trackify-brain.md`](brain-seed/trackify-brain.md) for the layout and sync command. When you
discover something non-obvious that future work needs (a decision, a trap, a fix to one of the live bugs
above), add or update a page under `brain-seed/` and run `bash brain-seed/sync.sh` — a **SessionEnd hook**
also runs it automatically, which is what keeps the brain current. Search it with `gbrain search "<keywords>"`
(embeddings are off, so use `search`, not `query`).
